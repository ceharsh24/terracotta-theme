Cross-Theme Summary & Contrast Ratios

01
Number / Constant: Collision Resolved
All themes now separate numbers from user constants using warm wine tones for numbers and amber for constants. Dark: #D890A0 vs #CC8858. Dark Dimmed: #D892A2 vs #C08050. Light: #7A3050 vs #944A10. Light Bright: #7E3454 vs #984E14. Numbers shift to wine hues, constants shift to amber, breaking the warm-orange corridor.
02
HTML Tag / Keyword: Separation Added
Light themes now use clay-rose tones for HTML/JSX tags (#8A4A5A, #8C4856), creating ~27 degrees of hue gap from keywords (#9A3016, #9E341A). In React code, <Badge> is now visually distinct from const.
03
Variable / Property: Semantic Lanes
Dark themes split variables (bright warm sand #CABFA6) from properties (soft blue #8DB5DD). Dark Dimmed uses #C2B89A / #88AED0. This warm/cool contrast creates clear semantic lanes for data containers vs member access.
04
WCAG AA Contrast: All Themes Pass
All syntax tokens meet WCAG AA (4.5:1) against their backgrounds. Comments are the lowest-contrast token at 4.7–5.1:1 (intentionally subdued). Keywords range 6.7–9.0:1, functions 7.4–8.3:1, strings 6.7–13.8:1. High Contrast CB achieves AAA across the board.
05
Python Decorators: Effective Across All Variants
Python decorators use a distinct purple shade across all themes (e.g., #CFB7E7 in Dark, #6036B0 in Light) and are always italicized. They provide instant visual landmarks in Python code.
06
Light Bright: Halation Fix Applied
Editor foreground softened from #141414 to #1E1E1E, reducing eye fatigue on the warm parchment #FAF9F4 background. The background itself was warmed from near-white to a subtle cream tone.
