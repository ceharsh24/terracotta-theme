Findings — Dark

WARN
String / Number collision: String #F0C24E and Number #D4B46A are both warm gold tones. In code with adjacent string literals and numeric constants (e.g., Java enums), these two token types are nearly indistinguishable.
WARN
Function over-saturation: Function color #2EE5B2 is extremely bright and saturated compared to all other tokens. It pulls excessive visual attention, especially in function-heavy Go and React code.
INFO
Annotation contrast: Java annotations and Python decorators #CFB7E7 provide good differentiation from classes #C28BFF — both purple-family but visually distinct.

Findings — Dark Dimmed

WARN
String / Number collision (inherited): String #DBAC3B and Number #C8A85C carry the same warm-gold collision as the Dark theme. Both remain hard to tell apart in mixed-literal contexts.
GOOD
Overall balance: The dimmed palette reduces saturation uniformly. Function color #3CB898 is much calmer than the Dark variant, and operator/keyword contrast feels well-tuned for extended reading.
INFO
Inherited class distinction: Inherited class #60A4E8 provides strong blue contrast against class purple #AF85E6 , making Java type hierarchies easy to scan.

Findings — Light

WARN
Variable / foreground merge: Variable color #3D3C44 is very close to the foreground #242424 . This makes it hard to distinguish code structure — variables blend into plain text, reducing scannability.
INFO
Parameter distinction: Parameter color #8D3500 provides excellent contrast against variable #3D3C44 and keyword #AB3D1E , making function signatures easy to read.
GOOD
Warm tone palette: The warm paper-like background #FDFAF6 gives a distinctive character. String #8E5C00 and number #B4135B are well separated (brown vs. magenta).

Findings — Light Bright

WARN
Variable / foreground merge: Same issue as Light — variable #30303A vs foreground #141414 are nearly identical dark tones on a white background.
WARN
Property / inherited collision: Property color #0349A4 is the same as inherited class #0349A4 . In Java code with both superclass references and property access, the two roles are indistinguishable.
INFO
Clean white canvas: Pure white background #FFFFFF with dark foreground provides maximum contrast for text readability, though the variable/foreground issue reduces token-level scannability.

Findings — High Contrast CB

GOOD
Excellent differentiation: All token types are clearly distinguishable. Bold keywords #FF8A5C , cyan functions #00E5FF , and purple classes #D580FF occupy distinct hue bands with no collisions.
GOOD
String vs Number resolved: String #FFDF33 (bright yellow) vs Number #FFB000 (warm amber) are well-distinguished — the hue gap is wide enough even for color vision deficiencies.
GOOD
Bold + underline modifiers: Using font-weight: bold for keywords and text-decoration: underline for inherited types adds a non-color channel, significantly improving accessibility.

Cross-Theme Summary & Recommendations

01
Strings vs Numbers: Warm Gold Collision
All dark variants have a warm gold collision between string and number tokens. Consider a cooler/greener tone for strings or a warmer orange for numbers to separate them. The Light themes solve this by using distinct hues (brown strings, magenta numbers). High Contrast CB resolves it with bright yellow vs amber.
02
Go: Missing Language-Specific Scopes
Go has no language-specific token rules (unlike Python, Java, or Rust). This means Go code relies entirely on generic TextMate scopes. Consider adding Go-specific scopes for chan, goroutine, defer, and interface method sets to improve semantic richness.
03
Light Themes: Variable Visibility
Both Light and Light Bright use variable colors (#3D3C44, #30303A) that are too close to their respective foregrounds (#242424, #141414). This reduces code structure visibility. Consider using a slightly different hue (warm brown or muted blue) for variables.
04
JSX/TSX: Tag vs Keyword Collision
Tag names and keywords share the same color across all themes. In component-heavy React code, JSX tags like <Badge> and JS keywords like const are visually identical. Consider a slightly different shade or saturation shift for JSX element tags.
05
Python Decorators: Effective Across All Variants
Python decorators use a distinct purple shade across all themes (e.g., #CFB7E7 in Dark, #6036B0 in Light) and are always italicized. They provide instant visual landmarks in Python code — this pattern works very well and should be preserved.
06
Java Improvements: Effective Across All Variants
Java-specific improvements — primitive type keywords, italic this, and distinct annotation coloring — are effective across all theme variants. These scoping refinements give Java code significantly more visual structure than default TextMate rules.
